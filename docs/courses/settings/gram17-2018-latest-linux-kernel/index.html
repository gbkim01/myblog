Gram17(2018년형) 최신 리눅스 커널 설치

  • 문제의 원인
  • 해결책

이 설정은 LG의 Gram17(2018년)형에 해당되는 내용입니다. 작년에 인터넷을 뒤지다가 국내 블로그에서 간신히 알아낸 설정 팁인데 출처가 어딘지 가물가물합니다. 검색해서 찾기도 어렵고… 기억나지 않는 그 분께 일단 감사말씀 드립니다.

정확하진 않지만 문제가 발생한건 우분투 18.04.02 버전인 것으로 기억합니다. 그 전 까지는 별다른 이상 없이 우분투 설치가 가능했는데 우분투 18.04.02부터 그램17에 설치하면 화면에 ACPI Error를 무한 출력하며 진행이 멈추는 현상이 발생했습니다.

문제의 원인

문제의 원인은 우분투 18.04.02 에 올라간 최신 리눅스 커널과 그램17의 썬더볼트 설정이 맞지 않은 것에 기인한다고 합니다. 따라서 바이오스에서 썬더볼트 설정만 만져주면 됩니다. 방법은 다음과 같습니다.

해결책

  • 부팅 시작될 때 F2를 눌러서 바이오스 진입

  • ctrl + alt + F7을 눌러서 고급 모드 진입(advanced mode)

  • Advanced 탭으로 이동 (방향키 오른쪽)

  • 맨 아래 Intel Advanced Menu 선택

  • Thunderbolt (TM) Configuration 선택

  • 맨 아래 Discrete Thunderbolt (TM) Configuration 선택

  • 맨 아래 Thunderbolt (TM) OS Select 선택

  • Windows 10 Thunderbolt support1를 Enabled나 Disabled로 변경

  • 자동 재시작 또는 F10을 눌러서 Save and Exit

  1. 기본값은 Enable + RTD3↩︎
